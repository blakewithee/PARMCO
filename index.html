Bluetooth Motor Control

An embedded system to control a Raspberry Pi-driven motor from an iPhone app via Bluetooth LE.

View on GitHub

System-Level Description

This project demonstrates a complete, end-to-end embedded system. The goal is to provide seamless wireless control over a physical motor from a custom-built mobile application.

Data & Control Flow

The flow of control begins with the user on the iPhone app. When a user adjusts the speed slider or taps a direction button, the app writes data to a specific characteristic on the BLE peripheral.

The Raspberry Pi, acting as the peripheral, constantly listens for these BLE characteristic updates. A C program running on the RPi receives the new data, parses it, and translates it into hardware commands.

These commands are sent to the L293D Motor Driver via the RPi's GPIO pins, managing both the speed (using PWM) and direction, which in turn drives the DC Motor.

Component Breakdown

The system is divided into three core components, each with a distinct responsibility.

1. iOS Application (The Controller)

The user-facing controller. This native app acts as the BLE Central device.

  • UI: Built with SwiftUI.
  • Function: Scans, connects, and sends commands.
  • Tech: Uses the CoreBluetooth framework.

2. Raspberry Pi 4 (The Brain)

The central hub and BLE Peripheral. It receives commands and manages hardware.

  • OS: Raspberry Pi OS.
  • Logic: A C program.
  • BLE: Uses BlueZ library (C).
  • Hardware: Controls GPIO pins via the pigpio C library.

3. Motor & Driver (The Muscle)

The physical hardware that translates digital signals into motion.

  • Driver: L293D H-Bridge.
  • Motor: Standard 12V DC Motor.
  • Function: H-Bridge handles direction and power.
  • Control: Speed managed by PWM signal.

Hardware Setup & Operation

A detailed look at the circuit design, pinout, and power requirements.

Circuit Operation

The circuit uses an L293D dual H-Bridge motor driver to control a 12V DC motor. This allows the Raspberry Pi to manage both the speed (via PWM to the `Enable` pin) and direction (via logic signals to `Input` pins).
An FC-51 IR proximity sensor is included for environmental input, such as obstacle detection. The sensor's digital `OUT` pin is fed to an RPi GPIO input.
An IRFZ34 N-Channel MOSFET acts as a low-side switch for an LED, which is tied to the motor's `Enable` pin to provide a visual status of when the motor is active.

Raspberry Pi Pin Assignments (BCM)

RP4 Pin (BCM) Function Description
GPIO 17 PWM Output Speed control to L293D `Enable` pin
GPIO 23 Direction 1 Motor direction to L293D `Input 1A`
GPIO 24 Direction 2 Motor direction to L293D `Input 2A`
GPIO 5 IR Sensor Input Reads digital output from FC-51
5V Logic Power Supplies 5V to L293D (VCC1) & sensor
GND Ground Common ground for all components

Power & Safety

  • The motor is powered by an external 12V DC supply.
  • The L293D logic (VCC1) and IR sensor are powered by the Raspberry Pi 5V pin.
  • A common ground between the RPi and the motor circuit is essential.
  • The L293D provides internal flyback diodes and thermal shutdown for protection.
  • Note: The FC-51 IR sensor must be connected to 5V, not 12V, to prevent damage.
