Hardware

Circuit Design & Setup

Detailed circuit schematic, pinout configuration, and power requirements

üîå Circuit Operation

The circuit utilizes an L293D dual H-Bridge motor driver to control a 12V DC motor, enabling the Raspberry Pi to manage both speed via PWM and direction via logic signals.

An FC-51 IR proximity sensor tracks RPM data from the motor, with its digital output fed to an RPi GPIO input for real-time monitoring.

An IRFZ34 N-Channel MOSFET acts as a low-side switch for an LED indicator, providing visual feedback when the motor is active.

‚ö° Power & Safety

  • ‚úì Motor powered by external 12V DC supply
  • ‚úì L293D logic and IR sensor powered by RPi 5V pin
  • ‚úì Common ground between RPi and motor circuit
  • ‚úì Internal flyback diodes and thermal shutdown protection
  • ‚ö†Ô∏è FC-51 IR sensor must use 5V, not 12V to prevent damage

Technical Stack

Code Libraries & Frameworks

A detailed breakdown of every library used in this project and the reasoning behind each choice

üì± iOS Application Libraries

SwiftUI

UI Framework

Apple's declarative UI framework for building native iOS interfaces with less code and real-time previews.

Why we used it: Modern, intuitive syntax allows rapid UI development. State management and reactive updates perfect for real-time motor control feedback. Native performance and seamless iOS integration.

CoreBluetooth

BLE Framework

Apple's framework for communicating with Bluetooth Low Energy devices as Central or Peripheral.

Why we used it: Industry-standard for BLE communication on iOS. Provides complete control over scanning, connecting, and GATT operations. Essential for wireless motor control implementation.

Combine

Reactive Framework

Apple's reactive programming framework for handling asynchronous events and data streams.

Why we used it: Elegant handling of BLE callbacks and state changes. Publishers and subscribers simplify data flow from UI to Bluetooth layer. Reduces callback complexity.

Foundation

Core Library

Fundamental framework providing essential data types, collections, and operating system services.

Why we used it: Required for Data types in BLE communication, UUID handling for GATT services/characteristics, and Timer functionality for periodic updates.

üñ•Ô∏è Raspberry Pi Libraries

BlueZ

Bluetooth Stack

The official Linux Bluetooth protocol stack. The system interacts directly with the BlueZ daemon to manage BLE connections.

Code Usage: Accessed via GDBus to register the GATT application and handle D-Bus method calls like WriteValue for incoming iPhone commands.

pigpio

Hardware Control

The core hardware interface library used for controlling GPIO pins with high precision.

Code Usage: Generates hardware-timed PWM for motor speed and uses gpioTick() to capture microsecond-precise timestamps for RPM calculations.

pthread

Concurrency

POSIX threading library used to run critical monitoring tasks in parallel with the main control loop.

Code Usage: Runs the RPM sensor monitoring in a separate thread to catch every blade pulse, using pthread_mutex to safely update shared data.

IPC (Named Pipes)

Inter-Process

Standard C library functions used to create a bridge between the BLE Server and Motor Control processes.

Code Usage: Uses mkfifo to create /tmp/motor_pipe, allowing the BLE service to send commands to the motor controller asynchronously.
